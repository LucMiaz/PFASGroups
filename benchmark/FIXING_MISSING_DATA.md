## Why Analysis Reports Tab Shows Nothing

The Analysis Reports page looks for JSON files in:
```
review-app/analysis_reports/
  ├── timing_analysis.json
  ├── complex_analysis.json
  └── enhanced_analysis.json
```

These files are generated by Python scripts that analyze the benchmark data. I've created placeholder files so the page doesn't error, but they need real data.

## Steps to Fix

### Option 1: On Your Linux System (Recommended - Has PFAS-Atlas)

```bash
# 1. Run OECD benchmark (important! ~4000 molecules)
cd /path/to/PFASGroups/benchmark
echo "2" | python enhanced_pfas_benchmark.py

# 2. Generate all analysis reports
./generate-analysis-reports.sh  # Or use the commands below

# 3. Copy files to Windows
# Copy these files from Linux to Windows:
#   data/pfas_oecd_benchmark_*.json
#   review-app/analysis_reports/*.json
#   review-app/analysis_reports/figures/*.png
#   review-app/analysis_reports/figures/*.svg
```

### Option 2: On Windows (If Python Works)

```powershell
cd c:\Users\luc\git\PFASGroups\benchmark

# Activate conda environment
conda activate chem

# Generate analysis reports
.\generate-analysis-reports.ps1

# This will:
# - Find latest benchmark files
# - Run analyze_timing.py
# - Run analyze_complex.py  
# - Run enhanced_analysis.py (if OECD exists)
# - Move files to review-app/analysis_reports/
```

### Option 3: Manual Analysis (If Python Doesn't Work on Windows)

If you can't run Python on Windows, run these on Linux then copy the files:

```bash
cd /path/to/PFASGroups/benchmark

# Run analysis scripts
python analyze_timing.py data/pfas_timing_benchmark_*.json
python analyze_complex.py data/pfas_complex_branched_benchmark_*.json
python enhanced_analysis.py data/pfas_enhanced_benchmark_*.json data/pfas_oecd_benchmark_*.json

# Move to review-app
mkdir -p review-app/analysis_reports/figures
mv *_analysis.json review-app/analysis_reports/
mv timing_*.{png,svg} complex_*.{png,svg} enhanced_*.{png,svg} review-app/analysis_reports/figures/
```
